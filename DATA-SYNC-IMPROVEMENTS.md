# 专项练习数据同步改进

## 改进日期
2026-04-04

## 改进目标
确保专项练习的答题结果正确同步到所有相关系统，包括：
- ✅ 练习历史记录
- ✅ 正确率统计
- ✅ 智能复习（SuperMemo算法）
- ✅ 错题本管理
- ✅ 学习进度更新

## 核心改进

### 1. 智能算法集成

**API端点**: `POST /api/v2/review/submit`

**SuperMemo SM-2算法质量评分标准**:
```
质量评分 0-5:
- 0: 完全忘记
- 1: 花时间但仍错误
- 2: 快速但错误（猜测）
- 3: 慢但正确
- 4: 正常速度且正确
- 5: 快速且正确
```

**自动计算质量评分**:
```javascript
// 答对情况
if (isCorrect) {
  if (timeSpent < 10秒)  quality = 5  // 快速且正确
  else if (timeSpent < 30秒)  quality = 4  // 正常速度
  else  quality = 3  // 慢但正确
}

// 答错情况
else {
  if (timeSpent < 30秒)  quality = 2  // 可能猜测
  else  quality = 1  // 花时间但仍错误
}
```

### 2. 错题本自动管理

**自动从错题本移除**:
- 质量评分 >= 4: 自动从错题本移除
- 保留SuperMemo学习记录
- 避免重复练习已掌握题目

**错题记录更新**:
- 质量评分 < 4: 更新复习间隔
- 计算下次复习时间
- 更新掌握度(mastery_level)

### 3. 数据同步流程

**答题后的数据流**:
```
1. 用户提交答案
   ↓
2. 判断正确性 + 计算质量评分
   ↓
3. 调用 /api/v2/practice/history
   - 记录练习历史
   - 更新用户统计数据
   ↓
4. 调用 /api/v2/review/submit
   - 应用SuperMemo算法
   - 更新ease_factor（难度因子）
   - 更新review_interval（复习间隔）
   - 更新mastery_level（掌握度）
   - 计算next_review_time（下次复习时间）
   - 质量评分>=4时自动从错题本移除
   ↓
5. 返回首页时触发数据刷新
   - 刷新统计卡片数据
   - 刷新错题本数量
   - 更新各练习模式统计数据
```

## 系统联动

### 1. 统计卡片更新
- **首页统计**: 总题数、已练习、正确率
- **错题本**: 待复习题目数量
- **学习进度**: 各类别进度更新

### 2. 智能复习同步
- **复习时间**: 根据遗忘曲线计算
- **优先级**: 即将遗忘的题目优先
- **掌握度**: 动态调整复习频率

### 3. 错题本管理
- **自动添加**: 答错时自动添加
- **自动移除**: 质量评分>=4时自动移除
- **智能排序**: 按复习时间和掌握度排序

## 数据一致性保证

### 原子操作
所有数据更新使用数据库事务，确保：
- 要么全部成功，要么全部回滚
- 不会出现数据不一致状态
- 练习记录和统计数据同步更新

### 错误处理
```javascript
try {
  // 1. 记录练习历史（核心）
  await axios.post('/api/v2/practice/history', {...})

  // 2. 更新智能复习（重要，失败不中断）
  try {
    await axios.post('/api/v2/review/submit', {...})
  } catch (reviewError) {
    console.error('智能复习失败:', reviewError)
    // 不影响继续练习
  }
} catch (error) {
  // 只记录错误，不中断练习流程
  console.error('记录失败:', error)
}
```
