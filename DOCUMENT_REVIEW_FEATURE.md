**实现日期**: 2026-04-01

## 📋 功能概述

按文档复习功能是一种新的学习模式，允许用户围绕原始法律法规和标准进行系统学习，避免知识碎片化，建立完整的知识体系。

---

## 🎯 核心特性

### 1. 文档分类体系

- **5级优先级分类**:
  - ⭐⭐⭐⭐⭐ 核心必考（出题概率 70%+）
  - ⭐⭐⭐⭐ 重要考点（出题概率 50-70%）
  - ⭐⭐⭐ 一般考点（出题概率 30-50%）
  - ⭐⭐ 补充考点（出题概率 10-30%）
  - ⭐ 其他（出题概率 <10%）

- **覆盖30+核心文档**:
  - 法律法规：密码法、网络安全法、数据安全法、个人信息保护法等
  - 技术标准：SM2算法、TLCP协议、SSLVPN/IPSecVPN等
  - 产品规范：服务器密码机、智能密码钥匙等
  - 检测认证：商用密码产品认证规则等

### 2. 智能学习统计

- 文档级别的学习统计
  - 总题目数
  - 已练习题目数
  - 正确率
  - 进度百分比

- 优先级分组统计
  - 按优先级汇总的学习进度
  - 整体准确率分析

### 3. 多维度筛选

- 按优先级筛选（1-5级）
- 按分类筛选（法律法规、技术标准等）
- 多种排序方式
  - 按优先级排序
  - 按正确率排序（优先推荐薄弱环节）
  - 按题目数量排序

### 4. 灵活的练习选项

- 选择每次练习的题目数量（5-100题）
- 排除已练习题目的选项
- 随机顺序练习
- 针对特定文档的集中练习

## 🔧 技术实现

### 数据库架构

新增字段到 `questions` 表：

```sql
ALTER TABLE questions
ADD COLUMN IF NOT EXISTS original_document VARCHAR(200),
ADD COLUMN IF NOT EXISTS document_category VARCHAR(100),
ADD COLUMN IF NOT EXISTS document_priority INTEGER DEFAULT 1;

-- 创建索引优化查询
CREATE INDEX IF NOT EXISTS idx_questions_original_document ON questions(original_document);
CREATE INDEX IF NOT EXISTS idx_questions_document_priority ON questions(document_priority DESC, original_document);
```

### 后端 API

文件：`/backend/document-review-api.js`

**5个核心端点**：

1. `GET /api/v2/documents` - 获取所有文档列表（带统计）
2. `GET /api/v2/documents/:documentName/questions` - 获取特定文档的题目
3. `GET /api/v2/documents/stats` - 获取文档复习统计
4. `GET /api/v2/documents/recommend` - AI推荐复习文档
5. `POST /api/v2/documents/update-metadata` - 批量更新文档元数据

### 前端组件

文件：`/frontend/src/components/DocumentReview.vue`

**主要功能**：

- 统计概览卡片
- 筛选和排序工具栏
- 按优先级分组的文档网格
- 文档卡片显示（图标、名称、统计、进度条）
- 练习选项弹窗
- 响应式设计（支持手机、平板、桌面）

### 导航集成

- **导航栏**: 添加了"📖 文档"按钮
- **首页**: 添加了"按文档复习"快捷卡片（带"推荐"标签）
- **路由**: `currentView === 'document-review'`

## 🚀 使用方式

### 用户操作流程

1. **访问文档复习页面**
   - 点击导航栏的"📖 文档"按钮
   - 或在首页点击"按文档复习"快捷卡片

2. **浏览文档列表**
   - 查看按优先级分组的文档
   - 查看每个文档的统计信息
   - 使用筛选器找到目标文档

3. **开始练习**
   - 点击文档卡片
   - 在弹窗中选择练习选项
   - 点击"开始练习"

4. **练习模式**
   - 系统提供该文档的题目
   - 可选择排除已练习的题目
   - 可设置每次练习的题目数量

### 管理员操作

**批量更新文档元数据**：

```bash
curl -X POST http://localhost:13000/api/v2/documents/update-metadata \
  -H "Content-Type: application/json"
```

这个操作会根据关键词自动为题目标注文档来源。
