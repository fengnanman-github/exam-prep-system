## 📊 一、新的分类体系设计

### 1.1 分类理念

**核心思想**：从学习者的角度设计分类路径，而不是单纯的知识分类。

**四个学习路径**：

1. **⚖️ 法律法规**（447题）
   - 密码法
   - 商用密码管理条例
   - 电子签名法
   - 电子印章管理
   - 电子认证服务
   - 关键信息基础设施
   - 重要会议精神
   - 违法线索处理

2. **📋 技术标准**（3125题）
   - 密码算法标准
   - 密码协议标准
   - 密码应用标准
   - 检测评估标准
   - 密码产品标准

3. **💼 应用实务**（775题）
   - 密码应用
   - 密钥管理
   - 密码产品
   - 密码检测
   - 案例分析

4. **🎯 综合知识**（728题）
   - 网络安全
   - 风险管理
   - 行业应用
   - 前沿技术

### 1.2 与旧体系对比

| 特性 | 旧体系 | 新体系 |
|------|--------|--------|
| 一级分类 | 13个法律法规 | 4个学习路径 |

## 🧠 二、基于遗忘曲线的智能练习算法

### 2.1 遗忘曲线配置

```
第1次复习：1天后
第2次复习：3天后
第3次复习：7天后
第4次复习：15天后
第5次复习：30天后
第6次复习：60天后
第7次复习：120天后
```

### 2.2 练习题目优先级

**优先级判断**：

| 优先级 | 条件 | 说明 |
|--------|------|------|
| ⭐⭐⭐⭐⭐ 5 | 错误率 > 50% | 急需复习 |
| ⭐⭐⭐⭐ 4 | 错误率 30%-50% | 建议复习 |
| ⭐⭐⭐ 3 | 到了复习时间 | 按时复习 |
| ⭐⭐ 2 | 掌握度 < 60% | 需要巩固 |
| ⭐ 1 | 已掌握 | 正常复习 |

### 2.3 新增 API 端点

| 端点 | 功能 |
|------|------|
| `GET /api/v2/practice/smart/:userId` | 智能推荐题目 |
| `POST /api/v2/practice/submit-with-srs` | 提交答案并更新复习计划 |
| `GET /api/v2/practice/daily-plan/:userId` | 获取今日复习计划 |
| `GET /api/v2/categories/learning-paths` | 获取学习路径分类 |
| `GET /api/v2/categories/subcategories` | 获取知识子类别 |
| `GET /api/v2/progress/:userId/learning-paths` | 获取学习路径进度 |

### 2.4 算法特点

1. **不重复做已掌握的题**
   - 连续3次做对的题目标记为"已掌握"
   - 已掌握题目优先级降为最低
   - 只在复习时间到达时才出现

2. **重点复习薄弱知识点**
   - 错误率高的题目优先出现
   - 最近做错的题目优先级更高
   - 自动识别薄弱知识点并强化练习

3. **新题智能推荐**
   - 优先推荐未做过的题目
   - 当没有需要复习的题目时，才推荐新题
   - 新题按照当前学习路径推荐

## 🎯 五、使用建议

### 5.1 学习路径建议

**第1阶段：基础知识**（1-2周）
- 从"技术标准"开始
- 重点学习密码算法标准和协议标准

**第2阶段：法律掌握**（1周）
- 学习"法律法规"路径
- 重点掌握密码法和商用密码管理条例

**第3阶段：应用实务**（1-2周）
- 学习"应用实务"路径
- 重点掌握密码应用和检测评估

**第4阶段：综合提升**（持续）
- "综合知识"路径贯穿始终
- 模拟考试查漏补缺

### 5.2 每日复习建议

1. **新题练习**：每天 20-30 道新题
2. **复习薄弱**：重点复习错误率 > 30% 的题目
3. **定时复习**：系统会自动安排复习提醒
4. **模拟测试**：每周 1-2 次模拟考试
