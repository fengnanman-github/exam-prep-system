# 密评备考系统 - 核心逻辑统一优化方案

## 一、当前问题分析

### 1.1 数据分散问题

**当前状态：**
- `practice_history` - 练习历史记录（按mode分开）
- `wrong_answers` - 错题本（带遗忘算法）
- `uncertain_questions` - 不确定的题目
- `user_progress` - 用户进度统计
- `exam_records` - 考试记录

**问题：**
- 不同练习模式的数据分散存储
- 遗忘算法只在错题本中使用
- 不确定标注独立存在
- 数据之间没有关联

### 1.2 核心逻辑不统一

**问题场景：**
1. 用户在"随机练习"中做了第1题，答错了
2. 切换到"分类练习"，第1题的状态没有同步
3. 切换到"文档复习"，第1题的状态又不一致
4. "智能复习"的推荐结果与其他模块不匹配

### 1.3 遗忘算法应用范围有限

**当前：**
- `spaced-repetition.js` 只在智能复习中使用
- `wrong_answers` 表有遗忘算法字段
- 但其他练习模式都没有使用这个算法

## 二、理想的核心逻辑架构

### 2.1 统一题目状态管理

每道题对每个用户只有**一个**状态记录：

```
QuestionState (题目状态表)
├── user_id: 用户ID
├── question_id: 题目ID
├── total_practiced: 总练习次数
├── total_correct: 总正确次数
├── total_wrong: 总错误次数
├── is_uncertain: 是否不确定（跨模式共享）
├── mastery_level: 掌握度 (0-1)
├── ease_factor: 难度因子 (SuperMemo 2算法)
├── interval: 当前间隔（天）
├── next_review: 下次复习时间
├── last_practice: 最后练习时间
├── first_practice: 首次练习时间
└── streak: 连续答对次数
```

### 2.2 统一的练习记录

所有练习模式都记录到同一个表，只是标记来源：

```
PracticeHistory (练习历史表)
├── user_id: 用户ID
├── question_id: 题目ID
├── source: 来源（random/category/document/exam/mock/smart）
├── user_answer: 用户答案
├── is_correct: 是否正确
├── time_spent: 花费时间
├── practiced_at: 练习时间
└── is_uncertain: 是否不确定
```

### 2.3 遗忘算法应用到所有推荐

**统一的推荐算法：**
```javascript
function getRecommendedQuestions(userId, options) {
  // 计算每道题的优先级分数

  // 基础分数（基于遗忘算法）
  let baseScore = calculateSpacingScore(questionState);

  // 调整因素
  if (question.is_uncertain) baseScore *= 1.5;  // 不确定的题目优先
  if (question.error_rate > 0.5) baseScore *= 2;  // 错误率高的优先
  if (question.mastery_level < 0.6) baseScore *= 1.3;  // 未掌握的优先

  // 考试权重调整
  baseScore *= getExamWeight(question.exam_category);

  return baseScore;
}
```

## 三、具体实施方案

### 3.1 数据库表结构调整

#### 方案A：扩展 wrong_answers 表（推荐）

将 `wrong_answers` 改名为 `question_states`，成为统一的题目状态表：

```sql
-- 重命名表
ALTER TABLE wrong_answers RENAME TO question_states;

-- 添加新字段
ALTER TABLE question_states ADD COLUMN IF NOT EXISTS:
  - total_correct INTEGER DEFAULT 0,
  - total_wrong INTEGER DEFAULT 0,
  - is_uncertain BOOLEAN DEFAULT false,
  - last_practice_at TIMESTAMP,
  - first_practice_at TIMESTAMP,
  - practice_count INTEGER DEFAULT 0,
  - source_modes TEXT[], -- 记录从哪些模式练习过
  - exam_category VARCHAR(100), -- 冗余字段，便于查询
  - law_category VARCHAR(100),
  - tech_category VARCHAR(100);

-- 创建索引
CREATE INDEX idx_question_states_next_review
  ON question_states(user_id, next_review_time);

CREATE INDEX idx_question_states_mastery
  ON question_states(user_id, mastery_level);

CREATE INDEX idx_question_states_uncertain
  ON question_states(user_id) WHERE is_uncertain = true;
```

#### 方案B：创建新表（更安全）

```sql
CREATE TABLE question_states (
  user_id VARCHAR(100) NOT NULL,
  question_id INTEGER NOT NULL,

  -- 统计数据
  total_practiced INTEGER DEFAULT 0,
  total_correct INTEGER DEFAULT 0,
  total_wrong INTEGER DEFAULT 0,
  current_streak INTEGER DEFAULT 0,

  -- 掌握度
  mastery_level NUMERIC(3,2) DEFAULT 0.0,
  ease_factor NUMERIC(3,2) DEFAULT 2.5,
  interval INTEGER DEFAULT 1,

  -- 时间相关
  next_review_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_practice_at TIMESTAMP,
  first_practice_at TIMESTAMP,

  -- 状态标记
  is_uncertain BOOLEAN DEFAULT false,
  is_favorite BOOLEAN DEFAULT false,

  -- 元数据（冗余，便于查询）
  exam_category VARCHAR(100),
  law_category VARCHAR(100),
  tech_category VARCHAR(100),
  document_name VARCHAR(200),

  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

  PRIMARY KEY (user_id, question_id),
  FOREIGN KEY (question_id) REFERENCES questions(id) ON DELETE CASCADE
);

CREATE INDEX idx_question_states_next_review
  ON question_states(next_review_time);

CREATE INDEX idx_question_states_mastery
  ON question_states(mastery_level);

CREATE INDEX idx_question_states_uncertain
  ON question_states(user_id, is_uncertain) WHERE is_uncertain = true;
```

### 3.2 统一的推荐算法API

创建新的推荐API：`/api/v2/recommendations/user/:userId`

```javascript
/**
 * 智能推荐题目
 *
 * 算法综合考虑：
 * 1. SuperMemo 2 遗忘算法
 * 2. 艾宾浩斯遗忘曲线
 * 3. 题目掌握度
 * 4. 不确定标注
 * 5. 考试类别权重
 * 6. 用户偏好
 *
 * 查询参数：
 * - mode: 练习模式 (random/category/document/exam/mock/smart)
 * - exam_category: 考试类别筛选
 * - law_category: 法律法规分类筛选
 * - tech_category: 技术专业分类筛选
 * - document_name: 文档筛选
 * - limit: 返回数量
 * - exclude_mastered: 排除已掌握题目
 * - only_uncertain: 只要不确定的题目
 */
router.get('/recommendations/user/:userId', async (req, res) => {
  const { userId } = req.params;
  const {
    mode = 'smart',
    exam_category,
    law_category,
    tech_category,
    document_name,
    limit = 20,
    exclude_mastered = false,
    only_uncertain = false
  } = req.query;

  // 构建查询
  const query = `
    WITH question_scores AS (
      SELECT
        qs.*,
        q.*,
        -- 计算推荐分数
        (
          -- 基础分数：基于遗忘算法
          CASE
            WHEN qs.next_review_time <= CURRENT_TIMESTAMP THEN 100
            ELSE 50 * EXP(-EXTRACT(EPOCH FROM (CURRENT_TIMESTAMP - qs.next_review_time)) / 86400)
          END *
          -- 考试权重
          CASE q.exam_category
            WHEN '密码应用与安全性评估实务综合' THEN 1.3
            WHEN '密码技术基础及相关标准' THEN 1.2
            WHEN '密码产品原理、应用及相关标准' THEN 1.2
            WHEN '密评理论、技术及相关标准' THEN 1.2
            WHEN '密码政策法规' THEN 1.1
            ELSE 1.0
          END *
          -- 不确定加成
          CASE WHEN qs.is_uncertain THEN 1.5 ELSE 1.0 END *
          -- 掌握度惩罚
          CASE
            WHEN qs.mastery_level < 0.4 THEN 1.5
            WHEN qs.mastery_level < 0.6 THEN 1.3
            WHEN qs.mastery_level < 0.8 THEN 1.1
            ELSE 0.8
          END
        ) as recommend_score
      FROM question_states qs
      JOIN questions q ON qs.question_id = q.id
      WHERE qs.user_id = $1
        ${exam_category ? 'AND q.exam_category = $2' : ''}
        ${law_category ? `AND q.law_category = $${exam_category ? 3 : 2}` : ''}
        ${tech_category ? `AND q.tech_category = $${exam_category ? (law_category ? 4 : 3) : 2}` : ''}
        ${document_name ? `AND q.original_document = $${exam_category ? (law_category ? 5 : 4) : (law_category || tech_category ? 4 : 3)}` : ''}
        ${exclude_mastered ? 'AND qs.mastery_level < 0.9' : ''}
        ${only_uncertain ? 'AND qs.is_uncertain = true' : ''}
    )
    SELECT *
    FROM question_scores
    ORDER BY recommend_score DESC
    LIMIT $${[exam_category, law_category, tech_category, document_name].filter(Boolean).length + 2};
  `;

  // 执行查询并返回
  const result = await pool.query(query, params);
  res.json(result.rows);
});
```

### 3.3 统一的答题提交处理

所有练习模式都调用同一个提交API：

```javascript
/**
 * 统一的答题提交API
 * POST /api/v2/practice/submit
 *
 * 自动更新：
 * 1. question_states 表（题目状态）
 * 2. practice_history 表（练习历史）
 * 3. user_progress 表（用户进度）
 * 4. uncertain_questions 表（不确定题目）
 */
router.post('/practice/submit', async (req, res) => {
  const {
    user_id,
    question_id,
    user_answer,
    is_correct,
    is_uncertain,
    source_mode, // random/category/document/exam/mock/smart
    time_spent
  } = req.body;

  const client = await pool.connect();

  try {
    await client.query('BEGIN');

    // 1. 更新或创建题目状态
    const stateResult = await client.query(`
      INSERT INTO question_states (
        user_id, question_id,
        total_practiced, total_correct, total_wrong,
        is_uncertain, last_practice_at,
        mastery_level, ease_factor, interval, next_review_time,
        exam_category, law_category, tech_category, document_name
      )
      VALUES (
        $1, $2, 1, $3, $4,
        $5, CURRENT_TIMESTAMP,
        0.0, 2.5, 1, CURRENT_TIMESTAMP + INTERVAL '1 day',
        (SELECT exam_category FROM questions WHERE id = $2),
        (SELECT law_category FROM questions WHERE id = $2),
        (SELECT tech_category FROM questions WHERE id = $2),
        (SELECT original_document FROM questions WHERE id = $2)
      )
      ON CONFLICT (user_id, question_id) DO UPDATE SET
        total_practiced = question_states.total_practiced + 1,
        total_correct = question_states.total_correct + $3,
        total_wrong = question_states.total_wrong + $4,
        is_uncertain = $5,
        last_practice_at = CURRENT_TIMESTAMP,
        -- 更新掌握度（使用SuperMemo 2算法）
        mastery_level =
          CASE
            WHEN question_states.total_practiced + 1 > 0 THEN
              (question_states.total_correct + $3)::numeric /
              (question_states.total_practiced + 1)
            ELSE 0
          END,
        -- 更新遗忘算法参数
        ease_factor =
          LEAST(2.5, GREATEST(1.3,
            question_states.ease_factor + (0.1 - (5 - $3) * (0.08 + (5 - $3) * 0.02))
          )),
        next_review_time = CURRENT_TIMESTAMP + (
          CASE
            WHEN $3 = true THEN
              (question_states.interval + 1) * question_states.ease_factor
            ELSE
              GREATEST(1, question_states.interval * 0.5)
          END
        ) * INTERVAL '1 day'
    `, [user_id, question_id, is_correct ? 1 : 0, is_correct ? 0 : 1, is_uncertain]);

    // 2. 记录练习历史
    await client.query(`
      INSERT INTO practice_history (
        user_id, question_id, user_answer, is_correct,
        time_spent, practice_mode, is_uncertain
      )
      VALUES ($1, $2, $3, $4, $5, $6, $7)
    `, [user_id, question_id, user_answer, is_correct, time_spent, source_mode, is_uncertain]);

    // 3. 更新用户进度
    await client.query(`
      INSERT INTO user_progress (user_id, total_practiced, total_correct, total_wrong)
      VALUES ($1, 1, $2, $3)
      ON CONFLICT (user_id) DO UPDATE SET
        total_practiced = user_progress.total_practiced + 1,
        total_correct = user_progress.total_correct + $2,
        total_wrong = user_progress.total_wrong + $3,
        updated_at = CURRENT_TIMESTAMP
    `, [user_id, is_correct ? 1 : 0, is_correct ? 0 : 1]);

    // 4. 处理不确定标注
    if (is_uncertain) {
      await client.query(`
        INSERT INTO uncertain_questions (user_id, question_id)
        VALUES ($1, $2)
        ON CONFLICT (user_id, question_id) DO NOTHING
      `, [user_id, question_id]);
    } else {
      await client.query(`
        DELETE FROM uncertain_questions
        WHERE user_id = $1 AND question_id = $2
      `, [user_id, question_id]);
    }

    await client.query('COMMIT');

    res.json({
      success: true,
      new_mastery_level: stateResult.rows[0].mastery_level,
      next_review_time: stateResult.rows[0].next_review_time
    });

  } catch (error) {
    await client.query('ROLLBACK');
    throw error;
  } finally {
    client.release();
  }
});
```

### 3.4 前端组件统一

所有练习组件使用相同的核心逻辑：

```javascript
// core-practice.js - 共享的练习逻辑
export const PracticeCore = {
  // 提交答案（所有模式共享）
  async submitAnswer({ userId, question, answer, mode }) {
    const response = await api.post('/api/v2/practice/submit', {
      user_id: userId,
      question_id: question.id,
      user_answer: answer,
      is_correct: answer === question.correct_answer,
      is_uncertain: this.isUncertain,
      source_mode: mode,
      time_spent: this.getTimeSpent()
    });

    // 更新本地状态
    this.updateQuestionState(question.id, response.data);

    return response.data;
  },

  // 切换不确定状态
  async toggleUncertain(questionId) {
    this.isUncertain = !this.isUncertain;
    // 实时同步到服务器
    await api.post('/api/v2/uncertain/toggle', {
      user_id: this.userId,
      question_id: questionId,
      is_uncertain: this.isUncertain
    });
  },

  // 获取推荐题目（所有模式共享）
  async getRecommendedQuestions(options) {
    const response = await api.get(`/api/v2/recommendations/user/${this.userId}`, {
      params: {
        mode: this.currentMode,
        ...options
      }
    });

    return response.data;
  }
};
```
